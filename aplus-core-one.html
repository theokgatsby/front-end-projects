Section 11: Wireless Networks
Wireless Frequencies
  • Direct Sequence Spread Spectrum (DSSS)
  • Suboptimal
  • Susceptible to electrical interference
  • Inefficent use of spectrum
  • Orthogonal Frequency Division Multiplexing (OFDM)
  • Most common
  • Minimizes interference and has higher rates of data
  • 2.4GHz band
  • Prone to interference from microwaves, bluetooth
  • To avoid overlap, use channels 1, 6, and 11
  • 5GHz band
  • Less interference, higher speeds, shorter range
  • 24 non-overlapping channels at 20MHz width
  • Channel bonding enables creation of wider channels
  • Combine adjacent channels for stronger signal
  • 6GHz band
  • Newer standard (WiFi 6e + 7)
  • Even less interference, higher speeds, shorter range
Wireless Standards
  • 802.11a - 5GHz, 54Mbps, OFDM
  • 802.11b - 2.4GHz, 11Mbps, DSSS
  • 802.11g - 2.4GHz, 54Mbps, OFDM
  • 802.11n - 2.4/5GHz, 600Mbps, MIMO, OFDM
  • 802.11ac (WiFi 5) - 5GHz, 1Gbps+, MU-MIMO, OFDM
  • 802.11ax (WiFi 6) - 2.4/5, 10Gbps+, OFDMA, MU-MIMO
  • 802.11be (WiFi 7) - 2.4/5/6GHz, 30Gbps+
  • MIMO (Multiple Input Multiple Output)
  • Multiple antennas for sending and receiving
  • Increases throughput and reliability
  • MU-MIMO (Multi-User MIMO)
  • Supports multiple devices simultaneously

Section 14: Network Configuration
DHCP
  • Automates process of assigning IP addresses
  • Scope is list of available IPs within a subnet
  • Leases are temporary IP assignments
  • Discover Offer Request Acknowledge (DORA)
  • Static assignments made for specific devices
DNS
  • Translates names to IP addresses
  • Think phone contact list
  • Mostly handled by ISPs and public DNS servers
  • Hierarchical structure with root, TLD, second-level domains
  • Subdomains are subdivisions of domains
  • Recursive lookup
    • Offloads all work to DNS resolver
    • Client requests IP from DNS resolver
    • Checks cache, root server, tld server, authoritative server
  • Interative lookup
    • More responsibility on client side resolver
    • Provides partial answers on how to find it
    • Refers to other DNS servers
    • Spreads the load
DNS Records
  • Address record (@ sign)
    • Link a domain name to an IPv4 address
  • AAAA record
    • Links to IPv6 address
  • CNAME record
    • Canonical name record
    • Aliases one domain name to another
    • Can only map to other domain names or subdomains
  • MX record
    • Mail exchange record
    • Specifies mail server for a domain
    • Can set priority
  • TXT record
    • Holds arbitrary text data for notes and comments
    • Can hold verification information
Spam Management with DNS
  • SPF (Sender Policy Framework)
  • Identities mail servers that are allowed to send emails for a domain
  • DKIM (DomainKeys Identified Mail)
  • Adds cryptographic signatures to verify email authenticity
  • Ensures emails have not been altered
  • DMARC (Domain-based Message Authentication, Reporting, Conformance)
  • Provide instructions for handling failed emails
  • User sends email with SPF and DKIM headers
  • Recipient server checks SPF, DKIM, and DMARC records
VLAN
  • Creates a logical separation of devices on a network
  • Improves security and reduces broadcast traffic
  • VLAN trunking allows multiple VLANs to share a single physical link
  • Native VLAN is the default VLAN for untagged traffic
VPN
  • Extends a private network over a public network
  • Site to site vpn
  • Connects entire networks together
  • Client to site vpn
  • Connects individual clients to a network
  • Clientless vpn
  • No hardware required, uses web browser
