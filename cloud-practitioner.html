Module 2 - Compute in the Cloud
Introduction to EC2
  • EC2 is flexible, cost-effective
  • On-demand capacity that's launched, scaled, terminated
  • Cost based on active usage
  • Multi-tenancy: Virtual machines are isolated but shares host resources
  • Choose instance type, operating system, storage to start
  • Divided into several instance familes
  • Three AMI's (Amazon Machine Image)
  • Create your own, pre-configured, purchase custom
EC2 Instance Types
  • General purpose
  • Diverse workloads
  • Web services, code repos, Uncertain workloads
  • Compute optimized
  • Gaming services, machine learning, scientific modeling
  • Memory optimized
  • Processing large datasets, data analytics, databases
  • Accelerated computing
  • Utilizes hardware accelerators like GPUs
  • Floating point calculations, graphics processing, machine learning
  • Storage optimized
  • For people needing high performance for locally stored data
  • Data warehousing, I/O intensive applications
How to Provision EC2
  • Control EC2 instance through API requests
  • Accessed by Management Console, CLI, or SDK
  • AWS Management Console
  • Web interface, access to services, search, workflows, monitoring
  • AWS CLI
  • Manage multiple services through command line
  • Automate tasks with scripts
  • AWS SDK
  • Programmatic access via various programming languages
  • AWS handles security of the cloud (hardware and infrustructure)
  • Customer handles security of data and applications
  • Unmanaged service, customer deals with firewall, OS, updates
EC2 Pricing
  • On Demand (Most popular)
  • No upfront cost, no commitment, pay for what you use
  • Savings Plan
  • Save by commiting to a consistent usage level for 1-3 years
  • Save up to 72%
  • Reserved Instances
  • Commit to 1-3 year term for predictable usage
  • Save up to 75%
  • Spot Instances
  • Bid on spare compute compacity
  • Save up to 90%
  • Dedicated Instances
  • Hardware dedicated to your account
  • Isolated from other AWS customers
Scaling Amazon EC2
  • Scale up by adding more power to machines
  • Scale out by adding more machines
  • Grows with your users or workloads
  • Scale resources up and down based on real time demand
  • Provides cost efficency and optimal resource usage
Load Balancing Traffic
  • Distribute incoming traffic across multiple resources
  • Prevents overload on any single instance
  • Instances are automatically added and removed to fit demands
  • Decouples front-end and back-end
  • Different routing methods
  • Round robin
  • Distributes in order
  • Least connections
  • Routes traffic to server with least active connections
  • IP Hash
  • Maps client's IP address to specific server
  • Least Response Time
  • Maps client's IP address to specific server
